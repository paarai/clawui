### GUI Environment Inheritance for CDP (04:00)

**Problem**: `launch_chromium_with_cdp()` failed to open browser windows when called from non-interactive shells (like OpenClaw exec) because it lacked `DISPLAY` and `XAUTHORITY`. Even though the user was logged into a graphical session, child processes could not connect to the X server.

**Solution**: Modified `skills/gui-automation/src/cdp_helper.py`:
- Added `inherit_gui_session_env()` that locates the user's graphical session process (gnome-session, Xorg, wayland) via `ps aux` and `loginctl`, then reads its environment from `/proc/<pid>/environ`.
- Extracts `DISPLAY`, `WAYLAND_DISPLAY`, `XAUTHORITY` and sets them in the current process before launching Chromium.
- Integrated into `ensure_gui_environment()` which now tries inheritance first, then fallback heuristics (socket detection).
- Also fixed missing `DEFAULT_USER_DATA_DIR` constant after earlier refactor.

**Validation**:
- When running from a terminal inside the graphical session, environment variables are correctly inherited: `DISPLAY=:0`, `WAYLAND_DISPLAY=wayland-0`, `XAUTHORITY=/home/hung/.Xauthority`.
- Headless Chromium launch already worked; this now enables headful mode (actual windows) from OpenClaw heartbeat/cron tasks.

**Test script**: `test_cdp_with_gui.py` prints environment and attempts to launch Chromium → navigate → screenshot.
